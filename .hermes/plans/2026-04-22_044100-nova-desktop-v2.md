# Nova Desktop v2 Plan

## Goal

Turn a functional but rough Electron wrapper into a polished, ship-worthy Pika desktop app that feels native and delightful on macOS.

## Current State (7 commits, ~550 LOC)

**What exists:**
- `BaseWindow` + `WebContentsView` architecture (good foundation)
- 72px sidebar with session tabs (initials, add/close/rename/switch)
- Session persistence to `sessions.json`
- Tray icon, dark mode, traffic light controls
- Deep link support (`pika://`), auto-retry on connection failure
- CI/CD: GitHub Actions → auto-release `.zip` for arm64 + x64
- Custom Pika rabbit icon (.icns + .png)

**What's broken or missing:**
- `node_modules/` is committed to git (no `.gitignore`)
- No window state persistence (size/position resets every launch)
- Sidebar has no visual indication of which session is loading
- No keyboard shortcut for switching tabs (⌘1-9)
- No right-click context menu in webviews
- Preload exposes minimal API — no notification bridge
- No auto-updater
- Unsigned app (Gatekeeper warning)
- Sidebar tooltip clips outside window bounds
- No loading/error states visible to user
- `package-lock.json` references only 2 deps (electron + electron-builder) — no runtime deps, which is correct

## Plan — 4 Phases

### Phase 1: Hygiene & Stability (do first)

| # | Task | Files | Why |
|---|------|-------|-----|
| 1.1 | Add `.gitignore` (node_modules, dist, .DS_Store, *.env) | `.gitignore` (new) | `node_modules/` is committed — 100+ MB of junk in git |
| 1.2 | Remove `node_modules/` from git history | terminal: `git rm -r --cached node_modules` | Clean the repo |
| 1.3 | Add `electron-store` for window state persistence (position, size, zoom level) | `main.js`, `package.json` | Window resets to 1280×860 center every launch — annoying |
| 1.4 | Persist active session ID so app reopens to last-used tab | `main.js` | Currently always opens to first session |
| 1.5 | Add proper error page (offline/load failure) instead of silent retry | `renderer/error.html` (new), `main.js` | User sees blank screen for 3s on failure with no feedback |
| 1.6 | Guard against `session.view.webContents.destroy()` race conditions | `main.js` | `close-session` IPC doesn't check `isDestroyed()` before operating |

---

## Recommended Execution Order

**Do Phase 1 first** — it's all stability/hygiene, no risk, immediate value. Especially 1.1+1.2 (gitignore) is embarrassing to leave.

**Phase 2 next** — this is what makes users say "this feels good." 2.1 (keyboard shortcuts) and 2.2 (right-click) are the highest-impact, lowest-effort wins.

**Phase 3** — notifications bridge (3.1) is the killer feature that justifies having a desktop app at all vs. just using the website.

**Phase 4** — code signing (4.1-4.2) requires an Apple Developer account + secrets setup. Auto-update (4.3) is the biggest unlock for distribution.

## Risks & Open Questions

1. **Code signing requires Apple Developer Program membership ($99/yr)** — does Pika have an org account we can use?
2. **Auto-updater needs code signing to work** — Phase 4.3 depends on 4.1
3. **pika.me CSP headers** — may block notification API override in preload. Need to test.
4. **Electron 34 is latest** — good, but verify `WebContentsView` API stability (it replaced `BrowserView` recently)
5. **Should sessions be independent cookie jars?** — currently they share a single Electron session, meaning login state is shared. This might be intentional (one login, multiple chat tabs) or a bug (can't have multiple accounts). Need to clarify product intent.
